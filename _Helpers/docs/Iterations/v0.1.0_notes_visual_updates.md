# Visual Style Notes

> **Usage** : Ce fichier doit être lu par Kilo Code avant toute modification dans `src/presentation/`.
> Il définit les règles inviolables pour le rendu visuel du MVP.

---

## 1. Identité visuelle

**Univers** : spatial, rétrofuturiste, doux et organique. "Synthwave cosy".
**Mot-clé de rendu** : formes rondes, lumières douces, pulsations lentes, profondeur suggérée sans 3D agressive.
**Références** : nébuleuses digitales, sphères lumineuses, halos éthérés.

---

| Rôle                    | Couleur             | Hex                   |
| ----------------------- | ------------------- | --------------------- |
| Fond                    | Noir violet profond | `#0a0612`             |
| Joueur                  | Cyan doux           | `#3fd6ff`             |
| Joueur (variante)       | Violet clair        | `#c084fc`             |
| Collectibles            | Rose/magenta doux   | `#ff80bf`             |
| Collectibles (variante) | Ambre doux          | `#ffd580`             |
| Anomalies               | Rouge désaturé      | `#c44040`             |
| Glow joueur             | Cyan clair          | `#8aeeff`             |
| Glow collectible        | Rose clair          | `#ffb3d9`             |
| HUD texte               | Blanc doux          | `#e8e0f0`             |
| HUD accents             | Cyan / violet       | `#3fd6ff` / `#9b59f5` |

**Règles couleur** :

- Pas de couleurs primaires pures (pas de `#ff0000`, `#00ff00`, `#0000ff`).
- Pas de blanc pur `#ffffff` pour les éléments de jeu.
- Les anomalies sont rouges mais toujours **désaturées** — jamais agressives.
- Les glows sont toujours plus saturés que la couleur de base, avec alpha entre 0.3 et 0.6.

## 3. Géométries autorisées

| Type           | Usage                           | Three.js         |
| -------------- | ------------------------------- | ---------------- |
| Sphère         | Joueur, collectibles, anomalies | `SphereGeometry` |
| Tore           | Anneaux de guidance, aura       | `TorusGeometry`  |
| Plan billboard | Halos, glows, trails            | `PlaneGeometry`  |
| Cercle flat    | Ripple de collecte              | `CircleGeometry` |

**Géométries interdites** :

- `BoxGeometry` (cube)
- `ConeGeometry`
- `CylinderGeometry`
- `IcosahedronGeometry`
- `OctahedronGeometry`
- Tout wireframe angulaire visible

**Règle absolue** : aucune forme avec des arêtes vives visible à l'écran.

## 4. Matériaux

### Orbe joueur

```ts
new THREE.MeshStandardMaterial({
  color: 0x3fd6ff,
  emissive: 0x3fd6ff,
  emissiveIntensity: 0.6,
  roughness: 0.4,
  metalness: 0.0,
});
```

### Orbe collectible

```ts
new THREE.MeshStandardMaterial({
  color: 0xff80bf,
  emissive: 0xff80bf,
  emissiveIntensity: 0.5,
  roughness: 0.5,
  metalness: 0.0,
});
```

### Glow billboard

```ts
new THREE.MeshBasicMaterial({
  color: 0x8aeeff,
  transparent: true,
  opacity: 0.35,
  depthWrite: false,
  blending: THREE.AdditiveBlending,
});
```

### Anomalie

```ts
new THREE.MeshStandardMaterial({
  color: 0xc44040,
  emissive: 0xc44040,
  emissiveIntensity: 0.4,
  roughness: 0.6,
  metalness: 0.0,
  transparent: true,
  opacity: 0.8,
});
```

**Règles matériaux** :

- `emissive` activé sur toutes les orbes et collectibles.
- Pas de `MeshPhongMaterial` avec shininess > 30.
- Pas de `MeshToonMaterial`.
- Les halos utilisent toujours `AdditiveBlending` avec `depthWrite: false`.

## 5. Caméra

- Type : `PerspectiveCamera`
- **FOV : 55°** (ne pas dépasser 65°, ne pas descendre sous 45°)
- Near : 0.1 / Far : 100
- **Position Z fixe : 8 unités** (vue "quasi-orthographique" — l'espace doit paraître plat et lisible)
- Pas de rotation de caméra (pas de pitch, pas de roll)
- Follow doux sur X/Y uniquement, avec **damping 0.88** (lent à rattraper le joueur)
- Légère anticipation optionnelle : la caméra regarde légèrement devant la direction du joueur

```ts
// Exemple de follow doux
camera.position.x += (target.x - camera.position.x) * (1 - 0.88) * dt * 60;
camera.position.y += (target.y - camera.position.y) * (1 - 0.88) * dt * 60;
// Z reste fixe
```

## 6. Éclairage

- **AmbientLight** : couleur `#1a0a2e` (violet très sombre), intensity 0.8
- **PointLight principal** : couleur `#6644aa`, intensity 1.2, distance 30, position (2, 3, 5)
- **PointLight secondaire** (optionnel, sur les collectibles proches) : couleur `#3fd6ff`, intensity 0.6, distance 8

**Interdits** :

- Pas de `DirectionalLight` avec des ombres activées (trop "3D hard").
- Pas de `SpotLight`.
- Pas de `HemisphereLight` à fort contraste.

---

## 7. Motion & animations

### Pulsation des orbes

- Fréquence : 0.8 Hz (lente, apaisante)
- Amplitude scale : ±4% (`baseScale * (1 + 0.04 * sin(t * 2π * 0.8))`)
- Easing : sine (naturel)
- Pas de pulsation > 2 Hz sur aucun élément

### Rotation

- Joueur : rotation Z lente (0.15 rad/s max), liée à la vélocité (pas de rotation constante)
- Collectibles : rotation Y très lente (0.2 rad/s), toujours la même vitesse
- Anomalies : légère oscillation de scale (pas de rotation rapide)
- Pas de rotation sur X (ça fait "3D aléatoire")

### Transitions de couleur / scale

- Toujours interpolées : `lerp(current, target, 0.1)` par frame
- Pas de switch de couleur instantané
- Pas de scale instantané

## 8. VFX : Ripple de collecte

Déclenché à chaque fragment collecté.

- Géométrie : `TorusGeometry` fin (tube radius 0.05)
- Couleur : couleur du collectible, `AdditiveBlending`
- Animation :
  - Scale : 0 → 2.5 en 600 ms, ease-out
  - Opacity : 1 → 0 en 600 ms, linéaire
- Pool de 5 ripples max simultanés
- Aucune autre géométrie créée dynamiquement (réutiliser le pool)

---

## 9. VFX : Trail du joueur

- Pool de 8 petites sphères (`SphereGeometry(0.12, 8, 8)`)
- Chaque sphère : `MeshBasicMaterial`, couleur joueur, `AdditiveBlending`
- Opacity de la tête du trail : 0.5, décroît vers 0 sur la queue
- Mise à jour : chaque élément du pool se déplace vers la position précédente du joueur (snake)
- Désactivé si vélocité < 0.05 (pas de trail au repos)

---

## 10. HUD

Le HUD est en DOM pur (pas de Three.js), superposé sur le canvas.

**Style général** :

- Police : monospace ou futuriste légère (ex : `'Courier New'`, `'Share Tech Mono'`, ou Google Fonts `'Orbitron'` avec fallback)
- Taille de base : 12–13px
- Couleur texte : `#e8e0f0`
- Fond des éléments : `rgba(10, 6, 18, 0.7)` (fond jeu semi-transparent)
- Bordures : `1px solid rgba(63, 214, 255, 0.3)` (cyan très transparent)
- Pas d'ombres portées lourdes
- Pas de gradients de fond complexes

**Éléments requis** :

- Titre "ECHO DRIFTER" en haut gauche, letterspacing élargi
- Indicateur layers : boutons DRONE / BASS / PAD / ARP / LEAD (5 total), état actif/inactif
- Jauge DISSONANCE : barre horizontale, couleur lerp cyan → rouge selon valeur
- Message contextuel bas d'écran (ex. "Drift toward a glowing fragment")
- Bouton RESTART en haut à droite, style sobre

## 11. Fond de scène

- Fond uni `#0a0612` via `renderer.setClearColor(0x0a0612, 1)`
- Optionnel (Phase 6 polish) : quelques points flottants très discrets (étoiles fixes)
  - 50–80 points max, `PointsMaterial`, size 0.05, opacity 0.4
  - Pas d'animation sur les étoiles au MVP
