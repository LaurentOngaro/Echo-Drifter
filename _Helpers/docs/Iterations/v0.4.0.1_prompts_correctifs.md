## Fix 2 — Background stars

In src/presentation/scene.ts, after the lights setup, add a static star field:
- 60 points using THREE.BufferGeometry + THREE.Points
- Positions: fixed values (NOT Math.random() — use a deterministic seeded
  distribution). Use this simple LCG seeder to generate positions:

  let seed = 42;
  function nextFloat() {
    seed = (seed * 1664525 + 1013904223) & 0xffffffff;
    return (seed >>> 0) / 0xffffffff;
  }

  Generate 60 points: x in [-30, 30], y in [-20, 20], z = -5 (behind gameplay).
- THREE.PointsMaterial: color 0xe8e0f0, size 0.04, transparent true, opacity 0.4,
  sizeAttenuation false.
- Stars are static: no animation, no update loop.
- Add stars to scene once at init, never update them.

## Fix 3 — Camera zoom control (viewSize)

In src/content/tuning.ts, visual.camera section, add:
  viewSize: 10,       // current default — orthographic visible height in world units
  viewSizeMin: 5,     // zoom in limit
  viewSizeMax: 20,    // zoom out limit
  viewSizeStep: 1,    // step per scroll tick

In src/presentation/scene.ts:
- Expose a function setViewSize(newSize: number) that:
  - Clamps newSize between visual.camera.viewSizeMin and visual.camera.viewSizeMax
  - Recalculates camera left/right/top/bottom with the new size
  - Calls camera.updateProjectionMatrix()
  - Stores the current viewSize in a local variable (mutable, initialized from tuning)

In src/main.ts:
- Listen for 'wheel' event on the canvas or window.
- On wheel down (deltaY > 0): call setViewSize(currentSize + step) — zoom out.
- On wheel up (deltaY < 0): call setViewSize(currentSize - step) — zoom in.
- currentSize is retrieved from scene via a getter getViewSize(): number
  exposed alongside setViewSize().
