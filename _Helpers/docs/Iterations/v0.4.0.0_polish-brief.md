## Fix P0 — Carré visible autour des orbes (glow billboard)

**Problème** : `PlaneGeometry` utilisé comme billboard glow affiche un carré
transparent visible autour des orbes, car le plan n'a pas de dégradé radial.

**Solution** : générer une texture de glow circulaire procéduralement via canvas 2D,
sans fichier image externe.

### Implémentation dans `src/presentation/meshes.ts`

Ajouter une fonction `createGlowTexture(color: number, size = 128): THREE.CanvasTexture` :

```ts
function createGlowTexture(color: number, size = 128): THREE.CanvasTexture {
  const canvas = document.createElement("canvas");
  canvas.width = size;
  canvas.height = size;
  const ctx = canvas.getContext("2d")!;
  const c = new THREE.Color(color);
  const hex = `#${c.getHexString()}`;
  const grad = ctx.createRadialGradient(size / 2, size / 2, 0, size / 2, size / 2, size / 2);
  grad.addColorStop(0, hex + "ff"); // centre : couleur pleine
  grad.addColorStop(0.4, hex + "88"); // mi-rayon : semi-transparent
  grad.addColorStop(1, hex + "00"); // bord : transparent
  ctx.fillStyle = grad;
  ctx.fillRect(0, 0, size, size);
  return new THREE.CanvasTexture(canvas);
}
```

Modifier `createPlayerGlow()`, `createCollectibleGlow()`, `createAnomalyGlow()` :

- Remplacer `MeshBasicMaterial({ color, ... })` par
  `MeshBasicMaterial({ map: createGlowTexture(color), transparent: true,
opacity: 1.0, depthWrite: false, blending: THREE.AdditiveBlending })`.
- Supprimer le `color` du material (la couleur est dans la texture).
- Passer `opacity: 1.0` — le dégradé gère lui-même la transparence bord.
- Taille du plan : garder `PlaneGeometry(1.6, 1.6)` pour le joueur,
  `PlaneGeometry(1.0, 1.0)` pour les collectibles.

## Fix P1 — Vue quasi-2D

**Problème** : FOV 55° avec des objets 3D donne encore une impression de
perspective. Le gameplay de drift nécessite une visibilité globale.

**Solution** : passer en `OrthographicCamera` dans `src/presentation/scene.ts`.
Ne pas toucher `src/gameplay/` — le gameplay reste en coordonnées XYZ, seule
la caméra change.

### Implémentation dans `src/presentation/scene.ts`

Remplacer `PerspectiveCamera` par `OrthographicCamera` :

```ts
const aspect = window.innerWidth / window.innerHeight;
const viewSize = 10; // unités visibles en hauteur — tunable
const camera = new THREE.OrthographicCamera(
  (-viewSize * aspect) / 2, // left
  (viewSize * aspect) / 2, // right
  viewSize / 2, // top
  -viewSize / 2, // bottom
  0.1, // near
  100 // far
);
camera.position.set(0, 0, 10);
camera.lookAt(0, 0, 0);
```

Ajouter dans `tuning.ts` section `visual.camera` :

```ts
orthographic: true,
viewSize: 10,   // hauteur visible en unités world — tunable
```

Mettre à jour `scene.ts` pour recalculer les bounds à chaque resize :

```ts
function resize() {
  const aspect = window.innerWidth / window.innerHeight;
  camera.left = (-visual.camera.viewSize * aspect) / 2;
  camera.right = (visual.camera.viewSize * aspect) / 2;
  camera.top = visual.camera.viewSize / 2;
  camera.bottom = -visual.camera.viewSize / 2;
  camera.updateProjectionMatrix();
  renderer.setSize(window.innerWidth, window.innerHeight);
}
```

Le `cameraRig.ts` n'a pas besoin de changer — il suit déjà X/Y avec Z fixe.

## Juice visuel — 3 effets à ajouter dans `src/presentation/vfx.ts`

### 1. Flash d'émission à la collecte

Déclenché via `spawnCollectFlash(position, color)` (nouveau dans `vfx.ts`).

- Sur l'orbe collectée : interpoler `emissiveIntensity` de 0.5 → 1.8 → 0.5
  sur 250 ms (pic à 80 ms).
- Implémenté via un timer interne dans le VFX system, pas dans `main.ts`.
- `main.ts` appelle simplement `vfx.spawnCollectFlash(pos, color)` au moment
  de la collecte.

### 2. Burst de particules à la collecte

Déclenché via `spawnBurst(position, color)` (nouveau dans `vfx.ts`).

- Pool de 12 petites sphères (`SphereGeometry(0.06, 6, 6)`),
  `MeshBasicMaterial`, `AdditiveBlending`.
- À l'activation : 6 sphères éjectées dans des directions régulières
  (tous les 60°), vitesse 2.5 unités/sec, friction 0.85/frame.
- Durée : 400 ms. Scale 1 → 0, opacity 1 → 0 sur la durée.
- Couleur = couleur du collectible.

### 3. Screen shake sur collecte et anomalie

Déclenché via `triggerShake(intensityUnits, durationMs)` (nouveau dans `vfx.ts`).

- Implémenté comme un offset XY sur la position de la caméra.
- Collecte : intensité 0.06 unités, durée 200 ms.
- Anomalie (dissonance > 0) : intensité 0.04, durée continue pulsée à 3 Hz,
  s'arrête quand dissonance = 0.
- Decay exponentiel : offset = intensité × exp(-t / durationMs) × bruit [-1,1].
- Le bruit est une simple valeur `Math.sin(t * fréquence)`, pas
  `Math.random()`, pour rester déterministe et doux.
- Exposé au `cameraRig.ts` via un getter `getShakeOffset(): THREE.Vector2`.

## Sons de contact — 3 sons à ajouter dans `src/audio/`

### Règle générale

- Tous les sons de contact restent dans la gamme A mineur pentatonique.
- `Math.random()` sur les Hz est interdit.
- Ajouter les paramètres tunable dans `tuning.ts` section `audio`.

### 1. Son de collecte enrichi (`playCollectTone`)

Le `playCollectTone` existant joue déjà une note triangle. Enrichir avec :

- Un deuxième oscillateur `sine` à la quinte supérieure (×1.5 la fréquence),
  gain 0.08, durée 180 ms.
- Un `convolverNode` ou `delayNode` 120 ms feedback 0.15 pour un léger écho.

### 2. Son de layer unlock (`playLayerUnlock`)

Nouveau dans `musicStateMachine.ts`, appelé automatiquement quand un layer
est activé via `unlock(id)`.

- Accord de 3 notes jouées en arpège rapide : root + tierce + quinte
  (A2 + C3 + E3 = 110 + 131 + 165 Hz).
- Oscillateur `sine`, chaque note 120 ms, espacées de 80 ms.
- Gain 0.15, attack 5 ms, release 200 ms.
- Filtre lowpass 1200 Hz.

### 3. Son de proximité d'anomalie (`playAnomalyProximity`)

Nouveau dans `musicStateMachine.ts`, appelé depuis `setDissonance(amount)`
quand `amount > 0`.

- Un LFO de volume à 4 Hz sur un oscillateur `sine` à 80 Hz (sous le drone).
- Gain proportionnel à `amount` : gain = `amount * 0.12`.
- Démarre avec fade-in 0.5 sec, s'arrête avec fade-out 0.8 sec quand
  `amount` revient à 0.
- C'est une tension basse fréquence, pas un bruit.
