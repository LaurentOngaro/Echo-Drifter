# v0.1.0.0_notes_audio_updates

> **Usage** : Ce fichier doit être lu par Kilo Code avant toute modification dans `src/audio/`.
> Il définit les règles inviolables de musicalité pour la génération audio procédurale du MVP.

---

## 1. Tonalité MVP

**Gamme unique** : A mineur pentatonique.

Notes autorisées (MIDI / fréquences de référence) :

| Note | MIDI | Hz (approx) |
| ---- | ---- | ----------- |
| A2   | 45   | 110 Hz      |
| C3   | 48   | 131 Hz      |
| D3   | 50   | 147 Hz      |
| E3   | 52   | 165 Hz      |
| G3   | 55   | 196 Hz      |
| A3   | 57   | 220 Hz      |
| C4   | 60   | 262 Hz      |
| E4   | 64   | 330 Hz      |

**Règle absolue** : aucune hauteur de note générée en dehors de cette liste au MVP.
`Math.random()` sur des fréquences ou des MIDI est **interdit**.

---

## 2. Timbres autorisés par layer

### DroneLayer (actif dès le début)

- 2 oscillateurs `sine`, légèrement désaccordés (+3 cents sur le second).
- Fréquences : root (A2 = 110 Hz) + quinte (E3 = 165 Hz).
- Filtre `lowpass` à 400 Hz, Q = 1.
- Gain master de la couche : 0.4.
- Pas de LFO rapide. Une légère modulation d'amplitude à 0.2 Hz (±5%) est autorisée.

### PulseLayer (actif dès le début, rejoint le drone après 2–3 sec)

- **Kick** : oscillateur `sine`, fréquence sweepée 80 Hz → 30 Hz en 80 ms, gain 0 → 0.8 → 0 (ADSR court).
- **Hat** : `white noise` filtré `highpass` à 6 kHz, durée 30 ms, gain 0.15 max.
- Tempo : 90 BPM fixes au MVP. Grid 4/4. Kick sur les temps 1 et 3, hat sur le 2 et 4.
- Pas de variation de tempo procédurale au MVP.

### PadLayer (débloqué par fragment 2)

- Oscillateur `triangle`, attack 1.5 sec, decay 0.3 sec, sustain 0.7, release 2 sec.
- Joue un accord simple en arpège très lent : A2 → C3 → E3, une note toutes les 2 secondes.
- Filtre `lowpass` à 800 Hz.
- Gain layer : 0.25.

### ArpLayer (débloqué par fragment 3)

- Oscillateur `sine` ou `triangle`.
- Séquence fixe de 4 notes dans la gamme : E3, G3, A3, C4 (loop).
- Vitesse : 0.5 note/sec (1 note toutes les 2 sec).
- Gain layer : 0.2.
- Légère réverb simulée par un `delay` 200 ms avec feedback 0.3.

### LeadLayer (débloqué par fragment 4)

- Oscillateur `sine`.
- Phrase fixe de 4 notes : A3 → C4 → E4 → A3, chaque note durée 0.8 sec.
- Pause de 1.6 sec entre les phrases.
- Gain layer : 0.18.
- Pas d'improvisation ni de génération procédurale de phrases.

### BassLayer (débloqué par fragment 1)

- Oscillateur `sine` ou `sawtooth` très filtré (lowpass 200 Hz).
- Une seule note : A2 (110 Hz), répétée sur le temps 1 de chaque mesure.
- Durée de note : 300 ms, ADSR simple.
- Gain layer : 0.35.

---

## 3. Interdits stricts

- **Pas de `Math.random()` sur des fréquences ou valeurs MIDI.**
- Pas d'oscillateur `square` brut à des fréquences > 500 Hz sans filtre lowpass agressif (< 300 Hz).
- Pas de `white noise` ou `pink noise` continu non filtré.
- Pas de plusieurs voix jouant simultanément des hauteurs non issues de la gamme A mineur pentatonique.
- Pas de glide/portamento entre des notes hors gamme.
- Pas de saturation / distortion au MVP.

---

## 4. Règles de densité

- **Démarrage** : DroneLayer seul pendant 2 sec, puis PulseLayer fade-in (1 sec).
- Chaque collectible n'active qu'un seul layer, dans cet ordre :
  1. Fragment 1 → BassLayer
  2. Fragment 2 → PadLayer
  3. Fragment 3 → ArpLayer
  4. Fragment 4 → LeadLayer
- Maximum 3 layers simultanés avant le fragment 3.
- Tous les fades in/out de layers : durée minimum 0.8 sec (pas de switch brutal).

---

## 5. Dissonance (anomalies)

- L'effet de dissonance est cosmétique, pas punitif.
- Implémentation : detune de +15 à +25 cents sur le DroneLayer uniquement.
- Simultanément : baisse du gain master de 20%.
- Durée max de l'effet : 3 sec si le joueur reste dans la zone.
- Résolution : retour progressif (fadeIn sur 1.5 sec après sortie de la zone).
- Pas de notes "fausses" supplémentaires ajoutées — seulement le detune et le gain.

---

## 6. Démarrage AudioContext

- Contrainte autoplay navigateur : afficher un overlay "Click to begin" au lancement.
- `AudioContext.resume()` uniquement après le premier input utilisateur (click ou keydown).
- Pas de tentative de démarrage audio avant ce premier input.
