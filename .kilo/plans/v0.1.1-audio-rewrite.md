# Echo Drifter — v0.1.1 Audio Patch Plan

## Goal

Rewrite the audio subsystem to match `_Helpers/docs/Iterations/v0.1.0_notes_audio_updates.md` exactly: a single in-scale tonality, deterministic frequencies, per-layer timbres specified in the brief, removed harsh warning, and a 4/4 grid pulse layer with kick + hat.

## Validated clarifications

- `src/audio/layers.ts` is **implicitly authorized** (omitted from allowed list in error). All edits remain in this plan file; actual code edits happen in a second pass after `plan_exit`.
- Anomaly warning = replaced by a soft low sine + delay (no square, no harshness).
- `playCollectTone` keeps `scaleHz(7 + count % 4)` (D5/E5/G5/A5 pentatonic), triangle, ~250 ms.

## 4. `src/audio/musicStateMachine.ts` (new)

Thin wrapper that owns:

- The `AudioContext`, `masterGain`, and the 6 layer instances.
- A small state table `current` ∈ { `idle`, `drone`, `drone+pulse`, `drone+pulse+bass`, `drone+pulse+bass+pad`, `…+arp`, `…+lead` }.
- Methods:
  - `start()` — schedules drone at t0+0, pulse at t0+2s (after drone solo).
  - `unlock(layer)` — for `bass/pad/arp/lead`, only triggers if preconditions match the fragment order; uses 0.8 s fade-in.
  - `setDissonance(amount)` — detune ONLY `drone` by `+20 cents` (or scaled 0→1 → 0→+20), set master × 0.8, ramp back over 1.5 s on exit.
  - `playCollectTone(freq)` — single triangle, ~250 ms.
  - `playAnomalyWarning()` — soft 80 Hz sine 30 ms + 200 ms delay feedback 0.2, every 600 ms; toggled by `setDissonance`.
  - `stop()` — fades all layers out 1.2 s and clears schedulers.

Composition: the layers are constructed in the state machine (not in `audioDirector`); `audioDirector` becomes a thin façade re-exporting the state machine's public methods to keep `main.ts` unchanged in shape.

## 5. `src/audio/layers.ts` — full rewrite

Each layer matches the brief literally:

### `DroneLayer` (extends `GainLayerBase`)

- 2 `sine` oscillators (`ctx.createOscillator` ×2, type `'sine'`).
- `osc1.frequency = rootHz` (110), `osc2.frequency = fifthHz` (165).
- `osc2.detune = +3 cents` (constant, set at init).
- `lowpass` filter: 400 Hz, Q=1.
- Amplitude modulation: `OscillatorNode` 0.2 Hz sine → `GainNode` ×0.05 → master output of layer (multiplies by 0.95→1.05). Wired to the layer's `gainNode`.
- Layer target gain = `audio.drone.layerGain` (0.4).
- `setDissonance(amount, when)` — adds `amount * 20 cents` detune to both oscillators (capped at 0→1), ramps over 0.3 s.

### `PulseLayer` (extends `GainLayerBase`)

- Scheduler: `setInterval(25ms)`, lookahead 0.25 s.
- Tempo: `audio.bpm = 90` → `secondsPerBeat = 60/90 ≈ 0.6667 s`.
- Grid 4/4: each beat = 1 quarter. Each "tick" = 1/2 beat (8th-note resolution).
- Kick on ticks 0 and 2 (beats 1 and 3): sine, freq `setValueAtTime(80) → exponentialRampToValueAtTime(30, +80ms)`, gain 0 → 0.8 → 0.0001 over 80 ms.
- Hat on ticks 1 and 3 (beats 2 and 4): 30 ms white noise burst highpassed at 6 kHz, peak 0.15.
- Layer target gain = `audio.pulse.layerGain`.
- No BPM drift under dissonance (brief: no procedural tempo variation).

### `BassLayer` (extends `GainLayerBase`)

- One note per measure on beat 1: A2 (110 Hz), sawtooth filtered lowpass 200 Hz (Q not specified, default 0.7).
- 300 ms duration with simple ADSR (attack 5 ms, decay 50 ms, sustain 0.4, release 200 ms).
- Layer target gain = `audio.bass.layerGain`.
- Uses `setInterval` only to re-schedule on `start()` (every bar = 4 beats).

### `PadLayer` (extends `GainLayerBase`)

- 1 triangle oscillator created/stopped per note.
- ADSR: attack 1.5 s, decay 0.3 s, sustain 0.7, release 2 s.
- Sequence: `A2 → C3 → E3` (110 → 131 → 165 Hz), one note every 2 s.
- Lowpass 800 Hz (Q default 1).
- Layer target gain = `audio.pad.layerGain`.
- Note duration = 1.8 s (slight overlap for smoothness).

### `ArpLayer` (extends `GainLayerBase`)

- Sine oscillator per note.
- Sequence: `E3 → G3 → A3 → C4` (165 → 196 → 220 → 262 Hz), 0.5 note/sec.
- One `DelayNode` (200 ms, feedback 0.3) shared between all notes, in series before `gainNode`.
- Layer target gain = `audio.arp.layerGain`.

### `LeadLayer` (extends `GainLayerBase`)

- Sine oscillator per note.
- Phrase: `A3 → C4 → E4 → A3` (220 → 262 → 330 → 220 Hz), each note 0.8 s, pause 1.6 s between phrases.
- Layer target gain = `audio.lead.layerGain`.

## 10. Risks & assumptions

- **Per-note oscillator creation** in Pad/Arp/Lead/Bass may produce many short-lived nodes. Acceptable for MVP; would pool later.
- **Drone amplitude LFO** (0.2 Hz ±5%) is wired as a constant modulation, not tempo-synced. Brief allows this.
- **Dissonance detune** is scaled linearly 0→1 → 0→+20 cents. Brief §5 says +15..+25; we use +20 as the max and scale smoothly.
- **Layer gain defaults not in brief** (e.g. `pulse.layerGain = 0.45`) are exposed in `tuning.ts` for easy tuning.
- **`scale.ts` is untouched**; we hard-code Hz in `tuning.ts` per the scope (drone root is 110 Hz not 220 Hz as `scale.ts` would produce with `ROOT_MIDI=57`).
