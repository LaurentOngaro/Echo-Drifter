# Echo Drifter — v0.1.2 Visual Patch Plan

## Goal

Rewrite the presentation layer so the visual output matches `_Helpers/docs/Iterations/v0.1.0_notes_visual_updates.md` exactly: rounded-only geometries, brief-specified materials/lights, fixed-camera "quasi-orthographic" view, deterministic 0.8 Hz pulses, lerped transitions, pure-DOM HUD with monospace font.

## Validated clarifications

- `src/main.ts` (wiring only) and `src/content/tuning.ts` (visual section only) are **implicitly authorized** to match the audio patch precedent. Edits to `tuning.ts` are restricted to a new `visual` export — palette, drift, camera, field, vfx, anomaly, audio blocks remain untouched.
- Halos around collectibles use **`TorusGeometry`** (thin tube, flat on XZ). No `RingGeometry` anywhere.
- Ground is kept as a **subtle `CircleGeometry`** at `y=-2`, opacity ≤ 0.15, color = background, no shadow.

## 2. `src/presentation/scene.ts` — full rewrite

Goals: clear color, fixed-camera rig, ambient + single point light, no fog (fog was removed because brief §11 forbids any background other than `setClearColor`).

```ts
- renderer.setClearColor(0x0a0612, 1)
- scene: no fog, no background texture
- camera: PerspectiveCamera(55, aspect, 0.1, 100); position (0, 0, 8); never touched on Z; never rotated.
- lights:
    AmbientLight 0x1a0a2e, intensity 0.8
    PointLight   0x6644aa, intensity 1.2, distance 30, position (2, 3, 5)
    (optional secondary PointLight 0x3fd6ff, intensity 0.6, distance 8 — only added if not redundant)
- ground: subtle CircleGeometry from `meshes.ts`, opacity 0.12, color = background, y = -2
- exposes: renderer, camera, scene, resize(), updatables[]
```

`createEchoScene` no longer takes velocity/look-ahead deps; it only sets up the stage. The camera follow logic is moved entirely to `cameraRig.ts`.

## 3. `src/presentation/cameraRig.ts` — full rewrite

Follow brief §5 exactly:

- Z position locked at `visual.camera.fixedZ` (= 8).
- X/Y damped toward `target.x, target.y` with `lerp(current, target, 1 - damping) = lerp(current, target, 0.12)` per frame.
- Optional micro-anticipation: `lookAt(target.x + vx * maxLookAhead, target.y, 0)` (no pitch).
- Damping is **frame-rate independent** by using `1 - exp(-(1-damping) * dt * 60)` (or simpler: stick to brief's example `(1 - 0.88) * dt * 60` for 60 fps target, plus a `Math.min(dt, 1/30)` cap to prevent jumps).

Interface preserved (caller passes `getTarget` and `getVelocity`); it returns an `Updatable`.

## 5. `src/presentation/vfx.ts` — full rewrite

### Ripple pool
- Max 5 concurrent ripples (brief §8).
- When spawning, if pool full → recycle the oldest.
- Geometry/material from `meshes.ts` `createRipple(color)`.
- Lifetime 600 ms.
- Scale: `startScale (0) → endScale (2.5)`, ease-out (cubic: `1 - (1-t)^3`).
- Opacity: 1 → 0 linear over 600 ms.

### Trail (player motion trail)
- Pool of 8 small spheres.
- Snake update: every 50 ms (or every frame at 60 fps), the tail reads the previous head's position; head reads current player position.
- If `velocity.length() < visual.trail.velocityThreshold` (0.05): all spheres alpha-fade to 0 over 200 ms, snake freezes.
- Opacity head = 0.5, linearly decays to 0 along the tail.

### Dissonance visual coupling
- `setDissonance(amount)` lerps the trail head opacity between 0.5 (calm) and 0.25 (anomaly); ripple opacity also dims.

### Public API preserved
- `updatable: Updatable`
- `spawnRipple(position, color)`
- `setDissonance(amount)` (unchanged)

## 12. Risks & assumptions

- **Google Fonts requires network**: acceptable for dev. Brief doesn't specify, but Orbitron is the safest default. If user wants fully offline, fallback stack already covers it.
- **Sprite vs Plane for glow**: brief mandates `MeshBasicMaterial`. Plane + per-frame `lookAt(camera)` chosen. Performance is trivial at 4 billboards.
- **Damping at variable framerate**: brief's example uses `(1 - 0.88) * dt * 60`. We add a `Math.min(dt, 1/30)` cap so a 5-second tab pause doesn't snap.
- **Trail performance**: 8 spheres is trivial, but the per-frame `lookAt(camera)` on each adds CPU. Acceptable for MVP.
- **Pulse interpolation**: implemented per-frame as `baseScale * (1 + amplitude * sin(t * 2π * 0.8))`. Looks "sine easing" without per-frame lerp.
- **`tuning.ts` add-only**: no existing export is renamed or removed, so `audio/`, `gameplay/`, `core/` continue to work unchanged.
