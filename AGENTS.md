# Echo Drifter — Agents Guide

## Mission

Build a portable MVP of Echo Drifter: a cosy-active synthwave exploration game where musical progression is a core mechanic.

## Rules

- Keep facts, assumptions, and design choices separated.
- Preserve portability to Godot/GDScript and Odin+raylib.
- Audio is a gameplay system, not decoration.
- Keep visuals rounded, organic, and pulsative.
- Avoid over-engineering.
- Prefer tunable config values over hardcoded constants.

## Workflow

1. Use plan mode for architecture and major feature changes.
2. Use code mode for implementation.
3. Use debug mode only for real bugs or regressions.
4. Update README when structure or commands change.
5. Summarize changed files and remaining assumptions after each phase.
